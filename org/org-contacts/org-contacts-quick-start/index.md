# org-contacts Quick Start

## Default properties

- EMAIL: The contact's email address(es) (multiple can be separated by commas).
- PHONE: The contact's phone number.
- BDAY: The contact's birthday. Formatted as YYYY-MM-DD (4-digit year required for agendas).
- ALIAS / NICKNAME: Alternate names to enable better auto-completion searching.
- ADDRESS: The contact's physical postal address.
- NOTE: Any freeform notes or additional context regarding the contact.

## Custom/Extended Properties

Users frequently extend org-contacts with custom properties to fit their specific networking or digital address book needs. Common examples include:

- COMPANY: The name of the company or organization the contact belongs to.
- URL: The contact's personal website, blog, or LinkedIn profile.
- ICON: A file path or link to an image/avatar of the contact.
- GROUP: Tags for categorizing contacts (e.g., Work, Family, Clients).
- CITY / STREET / ZIP: Granular address properties, typically used instead of one single ADDRESS field for vCard exports.
